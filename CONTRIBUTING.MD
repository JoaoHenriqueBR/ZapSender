# Contributing to ZapSender

First off, thank you for taking the time to contribute! It’s people like you who make open source such a great tool for everyone.

## 1. Contributor License Agreement (CLA)

To protect the project's future and allow us to maintain an "Open Core" model, we require all contributors to sign our Contributor License Agreement. 

**Why is this necessary?**
This agreement ensures that the project can remain open-source while allowing the maintainers to offer commercial versions or cloud-hosted options. 

**How to sign:**
1. Open a Pull Request with your changes.
2. Our **CLA Assistant** bot will automatically comment on your PR.
3. Follow the link in the comment to sign the agreement digitally (it takes 10 seconds).
4. Once signed, your PR will be cleared for review.

## 2. How Can I Contribute?

### Reporting Bugs
* Use the GitHub Issue Tracker.
* Describe the bug and include steps to reproduce it.
* Mention your OS (e.g., Nobara Linux, KDE Plasma version) and Python version.

### Suggesting Enhancements
* Open an issue to discuss the feature before starting to code.

### Pull Requests
* Create a new branch for your fix/feature.
* Ensure your code follows PEP 8 standards.
* **Include the AGPLv3 Header** at the top of any new `.py` files.

## 3. Style Guidelines
* We use **Black** for code formatting.
* Please write clear, descriptive commit messages.
